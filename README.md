# KBO Player Lab

KBO 선수를 선택하면 성적, 리그 내 위치, 성적 변화 이유, 연도/월별 변천사를 표와 그래프로 보여주는 선수 분석 웹페이지 프로젝트입니다.

## 목표

이 프로젝트는 단순 기록 조회가 아니라, 한 선수가 어떤 성적을 냈고 그 성적이 어떤 방식으로 만들어졌는지 설명하는 분석 리포트를 만드는 것을 목표로 합니다.

최종 사용자는 야구 데이터를 깊게 공부하지 않아도 다음 질문에 직관적으로 답을 얻을 수 있어야 합니다.

- 이 선수의 기본 성적은 좋은가?
- 리그 다른 선수들과 비교하면 어느 정도인가?
- 성적이 좋아지거나 나빠진 이유는 무엇인가?
- 연도별, 월별로 어떤 변화가 있었나?
- 수치, 표, 그래프, 이미지, 영상으로 볼 때 어떤 특징이 보이나?

## 현재 단계

현재는 1차 MVP 설계와 샘플 웹 프로토타입 단계입니다.

- 실제 데이터 수집 전
- 투수/타자 10명 샘플 데이터 기반
- 샘플 데이터 기반 화면 구성
- 향후 KBO, Statiz 등 공개 기록 데이터를 연결할 예정

## 폴더 구조

```text
kbo-player-lab/
  data/
    raw/          원본 데이터 CSV 저장
    processed/    정리된 분석용 데이터 저장
  docs/           프로젝트 기획, 로드맵, 작업 방식
  src/            데이터 수집/정리/분석 파이썬 코드
  web/            선수 분석 웹페이지
  outputs/        생성된 그래프, 리포트 결과물
```

## 첫 실행

아직 별도 서버가 필요 없는 정적 프로토타입입니다.

브라우저에서 아래 파일을 열면 됩니다.

```text
index.html
```

`index.html`은 실제 화면인 `web/index.html`로 자동 이동합니다.

## 현재 포함된 기능

- 이름 검색
- 팀명 필터
- 포지션 필터
- 선수 드롭다운 연동
- 타자/투수별 대표 리그 백분위
- 상세보기 영역
- 서번트형 상세 지표
- 연도별/월별 성적 흐름 전환
- 지표 선택 드롭다운
- 기본/상세/수비·WAR 기록표 전환

## 주의

현재 화면의 선수 수치는 실제 기록이 아니라 데모 데이터입니다.

실제 데이터 연결은 KBO 공식 기록과 Statiz에서 가져올 수 있는 항목을 확인한 뒤 진행합니다.

## 협업 방식

사용자는 방향과 판단을 제공합니다.

- 어떤 선수를 먼저 볼지
- 어떤 지표를 중요하게 볼지
- 설명 문장이 맞는지
- 화면이 이해하기 쉬운지
- 데이터 해석이 야구적으로 납득되는지

Codex는 구현과 분석 작업을 담당합니다.

- 데이터 구조 설계
- 크롤링/수집 코드 작성
- 지표 계산
- 표/그래프 생성
- 웹페이지 제작
- 설명 문장 초안 작성
- 수정 반영
