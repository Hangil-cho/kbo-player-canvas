# 로드맵

## Phase 0. 기준 잡기

완료 목표:

- 프로젝트 목표 정리
- 폴더 구조 생성
- 샘플 웹 프로토타입 생성
- 협업 방식 정리

현재 상태:

- 진행 중

## Phase 1. 샘플 데이터 기반 타자 분석 페이지

완료 목표:

- 샘플 타자 데이터 생성
- 선수 선택 기능 구현
- 핵심 지표 카드 구현
- 리그 평균 비교 영역 구현
- 연도별/월별 차트 구현
- 자동 해설 문장 초안 구현

사용자가 정해야 할 것:

- 첫 분석 선수 1명 또는 2명
- 타자부터 할지, 투수도 같이 고려할지
- 화면 톤: 진지한 리포트형, 팬페이지형, 대시보드형 중 선택

## Phase 2. 실제 데이터 연결

완료 목표:

- 데이터 출처 후보 조사
- 원본 CSV 수집
- 컬럼 구조 정리
- 기본 지표 계산 코드 작성
- 리그 평균/순위/백분위 계산

검토 대상:

- KBO 공식 기록
- Statiz
- 구단/선수 프로필
- 공개 기사/인터뷰

주의할 점:

- 웹사이트 이용약관 확인
- 과도한 요청 금지
- 데이터 출처 표시
- 수집 불가 데이터는 수동 입력 또는 링크 방식으로 처리

## Phase 3. 분석 로직 고도화

완료 목표:

- OPS 변화 원인 분해
- 출루율/장타율 기여도 분석
- 볼넷률/삼진율 변화 분석
- BABIP 등 보조 지표 추가
- 리그 평균 대비 강약점 자동 추출

예시 질문:

- 이 선수는 안타를 많이 쳐서 좋은가, 장타를 많이 쳐서 좋은가?
- 볼넷이 늘어서 좋아졌나?
- 삼진이 줄어서 안정적이 됐나?
- 리그 평균 대비 어떤 지표가 가장 강한가?

## Phase 4. 웹페이지 완성도 개선

완료 목표:

- 디자인 개선
- 모바일 대응
- 선수 검색/필터
- 표 정렬
- 그래프 설명 강화
- 선수별 리포트 공유용 HTML 생성

## Phase 5. 고급 데이터와 미디어 확장

완료 목표:

- 사진 영역 추가
- 영상 링크 또는 임베드 추가
- 타격폼/투구폼 변화 기록
- 구종/코스/타구 질 데이터 추가
- 기사/인터뷰와 수치 변화 연결

현실 체크:

- 타구속도, 하드힛, 회전수, 투구 움직임 같은 데이터는 공개 접근성이 낮을 수 있습니다.
- 확보 가능한 데이터부터 먼저 만들고, 고급 지표는 데이터 출처가 확인되면 확장합니다.
