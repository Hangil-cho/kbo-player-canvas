# 사용자가 해주면 좋은 일

이 프로젝트에서 사용자는 코딩을 직접 하지 않아도 됩니다.

대신 다음 역할을 해주면 프로젝트 품질이 훨씬 좋아집니다.

## 1. 방향 정하기

예시:

- 김도영부터 분석하고 싶어.
- 타율보다 OPS와 장타율 중심으로 보고 싶어.
- 초보자가 보는 페이지니까 지표 설명을 쉽게 해줘.
- 팬페이지 느낌보다 분석 리포트 느낌이 좋아.

## 2. 야구적 판단 주기

Codex는 데이터를 계산하고 해석 초안을 만들 수 있지만, 야구적으로 더 자연스러운 표현은 사용자가 잡아주면 좋습니다.

예시:

- 이 선수는 실제로는 부상 영향이 컸어.
- 이 해석은 너무 단정적인 것 같아.
- 타순 변경도 같이 봐야 할 것 같아.
- 단순히 부진이라기보다 역할이 바뀐 거야.

## 3. 데이터 출처 선택하기

사용자가 알고 있거나 신뢰하는 사이트를 알려주면 좋습니다.

예시:

- KBO 공식 기록을 우선으로 써줘.
- Statiz 지표도 같이 보고 싶어.
- 기사나 인터뷰는 링크만 걸어도 돼.

## 4. 결과물 피드백하기

화면과 문장을 보고 이런 식으로 말해주면 됩니다.

- 이 그래프는 이해가 어려워.
- 이 표는 너무 복잡해.
- 이 문장은 초보자가 이해하기 어려워.
- 이 부분은 더 야구팬스럽게 써줘.
- 이 선수의 장점이 더 먼저 보여야 해.

## 5. 다음 요청 예시

앞으로 Codex에게 이렇게 요청하면 됩니다.

```text
김도영을 첫 분석 선수로 해서 샘플 데이터를 실제 느낌에 가깝게 바꿔줘.
```

```text
이 페이지를 더 대시보드처럼 보이게 수정해줘.
```

```text
타율, 출루율, 장타율, OPS가 각각 무슨 의미인지 초보자용 설명을 추가해줘.
```

```text
Statiz에서 어떤 데이터를 가져오면 좋을지 조사해줘.
```

```text
선수 2명을 비교하는 화면도 추가해줘.
```
