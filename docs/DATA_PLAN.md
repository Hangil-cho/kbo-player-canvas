# 데이터 계획

## 데이터 우선순위

처음부터 모든 데이터를 모으지 않습니다.

1. 시즌별 기본 타격 기록
2. 월별 기본 타격 기록
3. 리그 평균 및 순위
4. 세이버 지표
5. 선수 프로필과 미디어
6. 고급 트래킹 데이터

## 확인한 데이터 출처 후보

- KBO 공식 기록실: 타자, 투수, 수비, 주루 공개 기록 표를 우선 확인합니다.
- KBO 스코어보드: 경기 일정, 점수, 이닝별 스코어를 보조 후보로 확인합니다.
- PTS/ABS 공식 또는 라이선스 제공자: 투구 추적 데이터는 계약 가능성이 확인된 뒤 사용합니다.
- 수동 CSV: 공식 기록에 없는 고급 지표와 기사/영상 근거를 보강합니다.
- Statiz, Naver Sports 숨은 API, 로그인 필요 문자중계는 자동 수집하지 않습니다.

현재 프로토타입의 수치는 데모 데이터입니다. 실제 연결 단계에서는 출처별 컬럼명과 계산 방식 차이를 먼저 문서화합니다.

## 수집 방식 원칙

- 공개 페이지에서 확인 가능한 기록부터 사용합니다.
- 과도한 요청을 보내지 않습니다.
- 출처와 수집일을 데이터 파일에 남깁니다.
- 수집이 애매한 고급 트래킹 지표는 `추정`, `수동 입력`, `링크 참조` 중 하나로 구분합니다.
- Statiz는 자동 크롤링하지 않습니다.
- Naver Sports 데이터는 정식 허락이나 API가 확인되기 전까지 사용하지 않습니다.
- WAR, RAA 등 계산 지표는 직접 계산, 수동 입력, 라이선스 데이터 중 하나로만 처리합니다.

## 1차 수집 구현

KBO 공식 기록 표 수집기는 아래 파일입니다.

```text
src/collect_kbo_official.py
```

CSV는 `data/raw`와 `data/processed`에 저장합니다. CSV는 GitHub에는 올리지 않고 로컬에서 수정 가능한 형태로 관리합니다.

필수 컬럼:

- plate_appearances

## 계산 가능한 지표

데이터가 충분하면 직접 계산할 수 있습니다.

- 타율 = 안타 / 타수
- 출루율 = (안타 + 볼넷 + 사구) / (타수 + 볼넷 + 사구 + 희생플라이)
- 장타율 = 총루타 / 타수
- OPS = 출루율 + 장타율
- 볼넷률 = 볼넷 / 타석
- 삼진율 = 삼진 / 타석
- ISO = 장타율 - 타율

## 비교 기준

선수 성적은 다음 기준과 비교합니다.

- 리그 평균
- 같은 포지션 평균
- 같은 팀 평균
- 규정타석 충족 선수 평균
- 전년 대비 변화
- 월별 변화

## 해석 주의사항

- 타석 수가 적으면 성적 변동이 클 수 있습니다.
- 월별 성적은 표본이 작아 과해석하면 안 됩니다.
- BABIP 상승은 운, 타구 질, 수비 위치 등 여러 원인이 섞일 수 있습니다.
- OPS는 직관적이지만 주루와 수비 가치를 모두 설명하지는 못합니다.
- wRC+, WAR 등은 출처별 계산 방식 차이가 있을 수 있습니다.
- PTS/ABS와 문자중계는 권리 확인 전까지 자동 수집하면 안 됩니다.
