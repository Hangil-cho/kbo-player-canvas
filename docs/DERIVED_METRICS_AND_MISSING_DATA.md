# 상세 지표 계산 및 미수집 데이터 정리

작성일: 2026-06-21

## 이번에 계산 반영한 지표

아래 지표는 현재 수집 중인 KBO 공식 기록 CSV만으로 계산해 웹 JSON에 반영한다.

### 타자

- `BB%`: 볼넷 / 타석
- `K%`: 삼진 / 타석
- `BABIP`: (안타 - 홈런) / (타수 - 삼진 - 홈런 + 희생플라이)
- `HR/PA`: 홈런 / 타석

해석 용도:

- `BB%`, `K%`, `BB/K`는 타석 접근, 선구안, 삼진 억제 흐름을 보는 보조 지표다.
- `BABIP`는 인플레이 타구 결과 변동을 보는 지표다. 타구 질, 수비 위치, 운이 섞이므로 단독 원인으로 해석하면 안 된다.
- `HR/PA`, `ISO`, `XBH`는 장타 생산 변화의 대체 지표로 쓴다. 타구속도와 배럴 데이터가 들어오기 전까지 장타 질을 간접적으로 본다.

### 투수

- `K%`: 탈삼진 / 상대 타자 수
- `BB%`: 볼넷 / 상대 타자 수
- `K-BB%`: K% - BB%
- `K/9`: 탈삼진 / 이닝 * 9
- `BB/9`: 볼넷 / 이닝 * 9
- `H/9`: 피안타 / 이닝 * 9
- `HR/9`: 피홈런 / 이닝 * 9
- `FIP`: (13*HR + 3*(BB+HBP) - 2*K) / IP + 리그 FIP 상수
- `BABIP Allowed`: (피안타 - 피홈런) / (상대 타수 - 탈삼진 - 피홈런 + 희생플라이)
- `LOB%`: (H + BB + HBP - R) / (H + BB + HBP - 1.4*HR)
- `Opponent OPS`: 상대 출루율 추정 + 상대 장타율 추정
- `IP/GS`: 이닝 / 선발 등판

해석 용도:

- `K%`, `BB%`, `K-BB%`는 구위와 제구의 가장 기본적인 결과 지표다.
- `FIP`, `HR/9`, `H/9`는 ERA가 왜 좋아졌거나 나빠졌는지 설명하는 보조 지표다.
- `BABIP Allowed`, `LOB%`는 실점 억제에 수비, 운, 상황 결과가 얼마나 섞였는지 볼 때 사용한다.
- `Opponent OPS`는 피안타율보다 넓게 상대 타자의 생산성을 보는 지표다.

## 계산하지 않은 지표와 이유

### 아직 계산하지 않는 가치 지표

- `WAR`
- `wRC+`
- `ERA+`
- `RAA`

이 지표들은 단순 공식 기록만으로도 일부 근사 계산은 가능하지만, 리그 평균, 파크 팩터, 포지션 보정, 대체 선수 기준, 득점 환경, 출처별 가중치가 필요하다. 지금 단계에서 임의 계산하면 신뢰도가 낮아질 수 있어 보류한다.

필요한 다음 작업:

- 우리가 직접 계산할 기준을 정한다.
- 파크 팩터를 제외한 간이 버전으로 시작할지 결정한다.
- 공식 또는 라이선스 데이터가 있으면 해당 기준을 우선한다.

### 현재 공개 공식 기록만으로 계산 불가한 트래킹 지표

- 타구속도
- 하드힛 비율
- 배럴 비율
- 발사각
- 구종 수
- 구종 비율
- 구종별 가치
- 구속
- 무브먼트
- 릴리스 포인트
- 팔각도
- 익스텐션
- CSW%, 헛스윙률, 존 컨택률

필요한 데이터:

- KBO/KBOP/공식 데이터 제공자의 PTS 또는 ABS 원천 데이터
- 투구 단위 또는 타구 단위 로그
- 각 투구의 구종, 구속, 위치, 결과
- 각 타구의 속도, 각도, 방향, 결과

사용자가 해야 할 일:

1. 상업용 웹 공개와 애드센스 사용이 가능한 데이터 제공처를 확인한다.
2. KBO, KBOP, 스포츠투아이 등 공식 또는 계약 가능한 제공처에 PTS/ABS 이용 가능 여부를 문의한다.
3. 데이터 샘플을 받으면 컬럼 정의서를 함께 확보한다.
4. 자동 수집이 아니라 파일 업로드 방식으로 먼저 붙일지 결정한다.

## 다음 추천 순서

1. 현재 계산한 파생 지표로 상세 분석 문장을 더 똑똑하게 만든다.
2. 월별 변화에서 `대표 지표 변화`와 `상세 지표 변화`를 나란히 비교한다.
3. 가치 지표는 먼저 간이 버전으로 설계만 하고, 실제 표시는 `베타` 또는 `계산 기준 필요`로 둔다.
4. 트래킹 지표는 라이선스 확인 전까지 UI에 `연결 예정` 상태로만 둔다.
